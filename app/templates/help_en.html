Multiple SSO/OAuth2 profiles

Admin → SSO supports multiple OAuth2/OpenID Connect profiles enabled at the same time. The login page displays one button for each active provider. The Google example pre-fills the standard Google OIDC endpoints and claims; administrators only need to add Client ID and Client secret and register the displayed redirect URI.

User documentation

Cybersecurity Incident Registry - complete user guide.

Download PDF

1. Overview

Cybersecurity Incident Registry supports the full lifecycle of cybersecurity incidents: creation, classification, action tracking, procedural warnings, notifications, reports, PDF form generation, full export and full import.

2. Incidents

The main page lists incidents visible according to the user's role. Writers and administrators can create and update incidents, associate personnel, categories, affected data, recommendations, documents and actions.

In the incident detail page the Procedural warnings section is displayed at the top, before the operational areas, so deadlines and missing requirements are immediately visible.

The maximum number of recommendations selectable for each incident is configured by administrators in Admin → Recommendations; the default value is 3 and is also enforced server-side.

3. Actions and deadlines

Actions are recorded with date, time, operator, label, description, consequences and attachments. Action labels can define a Maximum time (hours) and whether new actions with that label are exportable by default.

Automatic task-expiry notifications report dates and times using the application time zone configured by the administrator.

4. Notifications

Notification templates support placeholders and can generate emails for users, CSIRT, DPO and expiring tasks. Administrators configure SMTP, recipients, notification types and templates. Incident-level task notifications can be muted when needed.

5. PDF forms

The Forms menu is shown only when the user has privileges to access at least one of its entries. Administrators configure PDF templates, analyze AcroForm fields, map database fields to PDF fields and define font family and size used during PDF filling.

6. Reports, export and import

Users can produce incident reports and statistics. Full export/import includes incidents, users, configurable labels, settings, notification templates, PDF template configuration, audit records and binary files required by the application.

7. Language

The web interface and the user and administrator documentation are available in Italian and English. By default the browser locale is used: Italian is shown for Italian locale, English for all other locales. Administrators can force Italian or English from Admin → Other settings.

0.110-96 - User notification required

In incident Procedural warnings, the user-notification item is now worded as User notification required. This clarifies that the action must be recorded or generated through the notification workflow when required by the procedure.

8. Package README

The package now includes README_en.md as the English counterpart of README.md. Bilingual maintenance covers the interface, user and administrator documentation, the Italian README and the English README.

Incident-specific reminders

From the incident detail page, users can schedule reminders with an exact date and time, a custom message, and optional CC addresses. Primary recipients are the personnel associated with the incident.

After a restart, the scheduler recovers all due one-off reminders that have not been sent yet and records automated sends in the audit table.

Deadline notification scheduler

Periodic deadline notifications are executed by an internal scheduler that does not depend on web traffic. The functional interval remains the one configured in Admin → Notifications; every effective run is recorded in the audit log.

Compact Admin menu

The Admin menu is organized into collapsible subgroups and, on every page load, all subgroups are shown closed by default.

Concise audit log and release notes

The audit log collapses consecutive identical records: instead of creating many equal rows, it increments the occurrence count of the current record. Each block is closed at 100 occurrences and the counter then restarts on a new record.

Version notes are available from Help → Release notes, separate from the operational documentation.

Audit and CSV save

Administrators can browse audit records with search, pagination, total counts and the displayed interval. The Audit page can also save the current register as CSV and manage automatic or manual purge based on time retention and maximum row count.

Help menu and PDFs

The Help menu contains User documentation, Administrator documentation and Release notes. Direct PDF download entries have been removed from the menu; each PDF can still be downloaded from the button inside the corresponding page.

Duplicate-free deadline notifications

Deadline-task notifications are not resent multiple times within the same interval between two consecutive schedules: the application keeps the last successful send per incident and skips repeated polls in the same window.
