0.2.1-10 - External recipient search

  • The Admin and Settings address-book pages can now search external recipients by name, e-mail or notes.
  • The filter is preserved while editing, saving and deleting.
  • Italian and English documentation updated.

0.2.1-8 - Clickable workflow and procedural warnings

  • Procedural warnings are displayed immediately below expected operations.
  • Workflow step cards jump to the Actions section and preselect the associated label.
  • The form remains fully editable before saving.
  • Italian, English and project documentation updated.

0.2.1-6 - Workflows with descriptions and deadlines

  • Step captions use the task description, falling back to the task name.
  • Each step can have an additional operational description defined in the workflow.
  • Steps with a maximum completion time show limit, due time and remaining time.
  • The editable default workflow contains Initial information, Analysis, CSIRT notification, DPO notification and Closure.
  • Italian, English and project documentation updated.

0.2.0-7 - User removal

  • Added user removal from Admin → Users.
  • Removal deletes accounts and MFA tokens, while preserving incidents, reminders and audit records by detaching technical references.
  • Deletion of the current administrator and of the last remaining administrator is blocked.
  • Italian and English documentation updated.

0.2.0-4.2 - Uploaded SSO logo preview

  • Fixed graphical rendering of web-uploaded logos in the SSO logo storage list.
  • Shared logo previews now use the persistent configurable route for both default and uploaded logos.
  • User, administrator and project documentation updated in Italian and English.

0.2.0-127 - Production hardening

  • Automatic CSRF protection on all mutating forms.
  • HTTP security headers and secure cookie settings applied at application level.
  • Fail-fast secret validation when CIR_PRODUCTION=1.
  • Notification/reminder scheduler protected by PostgreSQL locking for multiple workers or replicas.
  • Docker Compose no longer contains hardcoded secrets and includes a new .env.example.

0.2.0 - Platform consolidation

  • Application version updated to 0.2.1-1, build 2026051901.
  • Bilingual user and administrator documentation updated with recent developments.
  • Consolidated anti-flooding audit, purge and CSV export, cron/interval notification scheduler, incident reminders, professional PDF reports, multi-profile SSO with shared logos and optional HTTPS/SSL.

0.1.0-125

  • Light grey SSO buttons on the login page.
  • Logo removal from shared SSO storage.
  • Removing a logo automatically clears it from associated SSO profiles.

0.112-12

Added multiple drag-and-drop tags on incident documents to automatically preselect manual notification attachments by notification type.

0.1.0-122

  • User documentation now includes application version/build/author information and clarifies that only exportable actions are used for document generation.

0.1.0-120

  • Added optional HTTPS/SSL support on container port 8443.

  • New Admin → HTTPS/SSL page to enable or disable HTTPS and upload the host certificate and private key.
  • Docker Compose, Kubernetes, full export/import and documentation updated.

0.1.0-119

  • SSO/OAuth2 now supports multiple configurable profiles active at the same time.
