  • Reviewed and fixed the possible sources of duplicate key value violates unique constraint on auto-increment primary keys.
  • Post full-import realignment now covers every application table with an id key, including workflows, scheduler states, template binaries, backups and future tables declared in SQLAlchemy metadata.
  • Full restore recreates the database and realigns all sequences after importing records with explicit IDs.
  • Documentation now describes the general duplicate-key protection.

  • The alignment also protects automatic LDAP/SSO user creation.
  • Documentation updated.
