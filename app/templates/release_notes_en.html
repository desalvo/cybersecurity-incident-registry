0.3.0-1 - Consequence rules and notification documents

  • AI Chatbot: added a global helpdesk/chat widget shown on every page when the plugin is enabled, with a desktop floating panel at the bottom right, a mobile button next to the menu and a minimize-back-to-icon control.
  • AI Chatbot: option to use a sanitized current-database snapshot, excluding personal/sensitive data, credentials, tokens, email addresses, binaries and potentially identifying free text.
  • Manual notification previews now preselect only documents with the matching notification tag, even when they were generated from a linked form template.
  • The automatic consequence logic under Admin → Other configurations can now add, remove and edit rules, text and conditions.
  • Conditions can use category, data type, severity, status, risk to rights/freedoms and textual fields.
  • Documentation updated.
  • Added single-workflow import/export with preview and selective overwrite confirmation.

  • Corrected the placeholder to %RECOMMENDATIONS% with automatic migration from %RECOMMENDATION%.
  • Added %APP_INFO% with application name, version and build.
  • The consequences field now combines configurable automatic consequences with consequences entered on actions.
  • Added the Automatic consequence logic section under Admin → Other configurations.

0.3.0-1 - Risk to rights and freedoms

The Personal data checkbox has been renamed to Risk to rights and freedoms. The matching workflow condition uses the same label, and deadline-task notifications compute %pending_actions% and %pending_actions_count% only from workflow steps applicable to the specific incident.
